# AI Work Judge

一个面向创业者和独立开发者的 `项目状态标记工具`。

它不再试图回答“这份 AI 作品写得怎么样”，而是回答一个更实际的问题：

`你手上的这个项目，现在到底还停留在消费态，还是已经开始进入生产态？`

---

## 项目简介

AI Work Judge 的核心目标是帮助用户持续追踪手上的项目，而不是沉迷于一次次“做了很多事”的幻觉。

当前版本将产品收缩成两层：

- `浏览器插件`
  - 在任意网页、GitHub 页面、Notion 页面或产品页面中，快速给当前项目打标签
- `极简 Web 后台`
  - 查看全部项目
  - 按状态筛选
  - 查看最近更新
  - 找出仍然值得继续推进的项目

这不是一个大而全的工作台，而是一个更轻、更快的小工具。

---

## 核心价值

- 快速给项目打标签，而不是写长报告
- 用 AI 提供建议，但保留人工最终判断
- 把项目从“感觉很忙”变成“状态可追踪”
- 帮创业者识别哪些项目还处于 `消费`，哪些已经开始 `生产`

---

## 状态体系

当前版本使用 4 个项目状态：

- `消费`
  - 当前更多是在看、学、想，还没有真实推进
- `转化中`
  - 已经有推进意图，但还没进入明确生产
- `生产`
  - 已经形成对外结果或明确推进
- `暂停`
  - 暂时不继续投入

AI 只会给出建议：

- `更像消费`
- `更像转化中`
- `更像生产`
- `建议人工判断`

最终标签始终由用户手动确认。

## 当前完成度

当前仓库已经包含以下能力：

### 1. 浏览器插件 MVP

位于 `extension/`：

- 读取当前标签页标题和 URL
- 自动识别来源类型，如 `github / notion / docs / web`
- 支持配置本地或线上后端地址
- 自动检查当前页面是否已经入库，并回显最近状态
- 请求本地 AI 建议
- 人工点击状态
- 写备注
- 保存到本地后端

### 2. Web 总后台

位于 `app/page.tsx`：

- 显示项目总数
- 显示 `消费 / 转化中 / 生产` 分布
- 项目状态筛选
- 待推进项目区
- 快捷标记项目状态

### 3. 极简后端记录

位于 `app/api/` 与 `lib/`：

- `GET /api/projects`
- `POST /api/projects`
- `POST /api/project-tags`
- `POST /api/project-ai-suggestion`

项目数据当前保存在：

- `data/projects.json`

## 为什么做这个项目

很多人使用 AI 后，项目数量变多了，但真正进入生产的项目并没有同步增加。

常见问题不是不会生成，而是：

- 项目越来越多，但不知道哪些值得继续
- 生成了很多内容，却没有形成真正推进
- 花了很多时间，却缺少一个持续判断项目状态的工具

AI Work Judge 想解决的是：

`如何把“我好像做了很多”变成“我知道哪些项目正在变成生产”。`

## 使用方式

### 本地运行 Web 后台

```bash
npm install
npm run dev
```

打开：

- [http://localhost:3000](http://localhost:3000)

### 加载浏览器插件

1. 打开 `chrome://extensions`
2. 开启 `开发者模式`
3. 点击 `加载已解压的扩展程序`
4. 选择仓库中的 `extension/` 目录
5. 保持本地服务运行
6. 首次打开插件时，可在弹窗中确认后端地址
7. 在任意网页中点击插件图标进行项目标记

### 线上部署建议

推荐部署到支持持久磁盘的服务，如 `Railway` 或 `Render`。

- 应用启动方式：直接使用 Next.js 默认启动命令
- 关键环境变量：
  - `DEEPSEEK_API_KEY`
  - `DEEPSEEK_API_BASE_URL`
  - `DEEPSEEK_MODEL`
  - `PROJECTS_DATA_PATH=/data/projects.json`

说明：

- 本地开发继续使用 `./data/projects.json`
- 线上环境把 `PROJECTS_DATA_PATH` 指向挂载磁盘路径，避免容器重启后项目记录丢失
- 插件里的“后端地址”改成你的线上域名后，就可以直接连线上后台

## 体验流程

最短体验路径如下：

1. 启动本地 Web 服务
2. 加载浏览器插件
3. 打开一个 GitHub 项目页或产品页
4. 点击插件图标
5. 查看 AI 建议
6. 手动选择状态并填写备注
7. 点击保存
8. 回到 Web 后台查看项目是否进入列表
